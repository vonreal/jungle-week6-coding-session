# 종료 처리 테스트 케이스 명세

이 문서는 `run_exit_tests.sh`에서 실행하는 에러/종료 처리 테스트를 설명합니다.

## 1) `exit_all_statements_success_exit_zero`
- 시나리오:
  - 전체 SQL 문장을 모두 성공시키고 실행합니다.
- 의도:
  - 에러가 없을 때 종료코드가 `0`인지 확인합니다.
- 검사:
  - 종료코드 `0`
  - `stderr` 비어 있음
  - `stdout`은 정상 SELECT 결과

## 2) `exit_any_error_makes_exit_one`
- 시나리오:
  - 중간에 1개 에러가 발생하는 SQL 파일을 실행합니다.
- 의도:
  - 에러 1건 이상이면 종료코드가 `1`로 집계되는지 확인합니다.
- 검사:
  - 종료코드 `1`
  - `stderr`에 에러 1줄 출력
  - `stdout`은 기대 결과와 일치

## 3) `exit_mixed_errors_keep_running_and_exit_one`
- 시나리오:
  - 서로 다른 에러를 포함한 문장들과 정상 문장을 섞어 실행합니다.
- 의도:
  - 에러 발생 후에도 다음 문장을 계속 실행하고 최종 종료코드는 `1`인지 확인합니다.
- 검사:
  - 종료코드 `1`
  - `stderr`에 문장별 대표 에러가 순서대로 출력
  - `stdout`은 정상 문장 결과만 포함

## 4) `exit_one_error_message_per_statement`
- 시나리오:
  - 한 문장에서 다중 문제가 있을 수 있는 실패 입력을 실행합니다.
- 의도:
  - 한 SQL 문장당 대표 에러 1개만 출력하는지 확인합니다.
- 검사:
  - 종료코드 `1`
  - 해당 문장에 대해 `stderr` 에러 1줄만 출력
