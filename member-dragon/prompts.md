# AI 활용 이력

## 사용 도구
- Codex (GPT-5 계열 코딩 에이전트)
- Codex 서브 에이전트 1개로 하네스 문서 정리
- Codex 서브 에이전트 1개로 구현 후 코드 리뷰 병렬 요청

## 프롬프트 기록

### 1단계: 뼈대 구축
> `readme.md 를 읽어보고 common 폴더 내부를 먼저 파악한다음 내 담당인 member-dragon 폴더 내부를 한번 확인해봐`

**AI 응답 요약**: 루트 `README.md`, `common/`, `member-dragon/`를 읽고 현재 과제 범위와 폴더 상태를 정리했다.
**내 판단**: 구현 전에 공통 규약과 내 폴더 상태를 먼저 맞추는 단계로 채택했다.

### 2단계: 파서 구현
> `codex 구현을 위한 plan_codex.md 에 계획을 만들어놨어 먼저 한번 검토해보고 어떻게 이해했는지 나한테 알려줘`

**AI 응답 요약**: `plan_codex.md`를 검토하면서 `ResultSet` 메모리 소유권, 스키마 로드 실패 처리, statement 분리, INSERT 컬럼 목록 처리의 문제를 짚었다.
**내 판단**: 단순 요약이 아니라 실제 구현 전에 수정해야 할 설계 결함을 찾는 데 도움이 되어 채택했다.

### 3단계: 하네스 적용 + 구현
> `검토결과에서 문제가 되는점은 수정을 하되 왜 수정했는지 뭐가 문제인지 기록해둬 . 그리고 하네스 프로그래밍을 적용하려고 하네스 스타터팩을 만들어서 폴더로 넣어놨는데 이것도 적용해서 하네스프로그래밍을 해보고 싶은데 서브 에이전트나 추가 에이전트가 필요하면 10개 이내로 추가해서 병렬로 움직여도 괜찮으니까 하네스 프로그래밍 적용해서 이번에 구현을 해보자`

**AI 응답 요약**: 하네스 문서와 로그를 갱신하고, 병렬 문서 정리와 함께 `member-dragon/src/` 구현, 공개/숨은 테스트 통과, 기록 문서 업데이트까지 진행했다.
**내 판단**: 이번 구현의 핵심 지시였고, 실제 산출물과 로그까지 남겨야 했기 때문에 그대로 채택했다.

### 4단계: 파일 기반 저장 요구사항 재정렬
> `요구사항이 이거라서 파일로 결과를 저장해야하는데 플랜이 이상했네`

**AI 응답 요약**: 현재 구현이 메모리 저장이라는 점을 인정하고, vtable 구조는 유지한 채 `file_storage.c`로 교체해서 실제 파일 기반 DB 요구사항에 맞추는 방향으로 수정했다.
**내 판단**: 테스트 통과보다 원 과제 요구사항이 우선이므로 저장 계층을 파일 기반으로 바로 수정하는 쪽을 채택했다.

### 4단계: 구현 후 독립 리뷰
> `member-dragon/src`만 기준으로 공통 명세와 테스트에 맞는지 집중 코드 리뷰를 해 달라.

**AI 응답 요약**: 구현 후 사양 불일치, 리스크, 빠진 검증이 있는지 별도 에이전트 리뷰를 병렬 요청했다.
**내 판단**: 하네스의 review 단계를 채우기 위한 독립 확인이라 유지했다. 최종 정리에는 이 결과를 반영한다.
