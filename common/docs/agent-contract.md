# 공통 에이전트 계약문서

## 목적
개인 구현 결과를 공정하게 비교하기 위해, 공통 자산과 개인 작업 범위를 엄격히 분리한다.

## 건들면 안 되는 것
`common/` 폴더 전체는 수정 금지.

- 테스트 케이스
- 스키마
- 채점 스크립트
- IO 스펙 문서

이 영역을 개인별로 수정하면 결과 비교가 불가능해진다.

## 건들어야 하는 것
각자 `member-본인이름/` 폴더 안에서만 작업한다.

- `src/`에 코드 작성
- 폴더 루트에 산출물 문서 작성

## 필수 산출물 4개
1. `src/`
- 컴파일/실행 가능한 SQL 처리기 코드
- `Makefile` 포함

2. `prompts.md`
- AI에게 보낸 프롬프트 기록
- AI 결과를 채택/수정/기각한 판단 근거

3. `decisions.md`
- 저장 포맷
- 파서 구조
- 에러 처리 범위
- 기타 설계 판단과 근거

4. `trouble.md`
- 삽질/실패 사례
- AI 제안의 오류와 수정 과정
- 최종 해결 방법 및 배운 점

## 테스트 방법
개인 코딩 시간에는 공개 테스트만 실행한다.

```bash
./common/scripts/run_tests.sh ./member-본인이름/src/sql_processor public
```

히든 테스트는 Phase 3 발표 시점에 처음 실행한다.
개인 구현 단계에서는 공개 테스트 6개 통과를 목표로 한다.
