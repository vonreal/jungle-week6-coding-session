# 삽질 로그

## 이슈 1: PowerShell에서 한글 테스트 파일이 깨져 보임
- **상황**: 공통 문서와 테스트 SQL을 확인할 때 한글이 깨져 보여 내용을 잘못 해석할 위험이 있었다.
- **원인**: PowerShell 기본 출력 인코딩과 파일 인코딩 표시 방식 차이 때문이었다.
- **AI가 준 답**: UTF-8로 다시 읽는 명령을 사용해 문서 내용을 재확인하자고 제안했다.
- **실제 해결**: `System.IO.File::ReadAllText(..., UTF8)` 방식으로 문서를 재확인해서 계약 문서와 IO 스펙을 정상 해석했다.
- **배운 것**: Windows 터미널에서 파일 내용을 읽을 때는 인코딩 확인을 먼저 해야 한다.

## 이슈 2: 로컬 PowerShell 경로에서 `gcc`, `make`가 바로 잡히지 않음
- **상황**: 구현 후 즉시 로컬 빌드를 확인하려 했는데 `gcc`, `make` 명령이 현재 셸에서 인식되지 않았다.
- **원인**: 현재 PowerShell PATH에 컴파일러/빌드 도구가 직접 연결되어 있지 않았다.
- **AI가 준 답**: 구조와 구현을 먼저 진행하고, 마지막에 사용 가능한 빌드 경로(`bash`, `wsl`, 다른 컴파일러`)를 재확인하자고 제안했다.
- **실제 해결**: 우선 코드 구조와 문서를 완성하고, 이후 사용 가능한 빌드 경로를 별도로 점검하는 방식으로 작업 순서를 조정했다.
- **배운 것**: 환경 확인은 초기에 하고, 막히면 구현과 검증 단계를 분리하는 게 진행에 유리하다.

## 이슈 3: 체크리스트 검증 중 테스트 입력 컬럼이 스키마와 불일치
- **문제**: 체크리스트 2번 검증 초기에 `users` 테이블에 없는 컬럼(`id`, `email`)을 사용해 실행이 실패했다.
- **원인**: 검증용 SQL을 작성할 때 실제 `users.schema`의 컬럼(`name`, `age`, `major`)을 기준으로 맞추지 못했다.
- **해결**: 검증 SQL을 스키마 기준 컬럼으로 수정하고 동일 항목을 재실행해 정상 통과를 확인했다.
- **재발방지**: 새 검증 케이스 작성 전 `common/schema/*.schema`를 먼저 확인한 뒤 SQL을 작성한다.
