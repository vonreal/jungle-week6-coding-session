# AI 활용 이력

## 사용 도구
- Codex (GPT-5 계열 코딩 에이전트)

## 프롬프트 기록

### 1단계: 과제 해석과 작업 범위 확인
> common에 있는 agent-contract.md를 먼저 확인해주세요. 그리고 io-spec.md도 확인 해주세요. 그리고 제가 건들면 안되는 부분과 구현해야 하는 파일에 대해 설명해주세요. 참고로 저는 jihyun입니다.

**AI 응답 요약**: 공통 문서를 읽고 `common/`은 수정 금지, 작업은 `member-jihyun/` 안에서만 해야 한다는 점을 정리했다.
**내 판단**: 채택. 과제 계약을 먼저 고정해야 이후 구현 방향이 흔들리지 않는다.

### 2단계: 목표 프로젝트 가능성 검토
> 제가 만들고 싶은 프로젝트는 “파일 기반 Mini SQL Processor + Page 관리 + 간단한 Buffer Cache” 입니다. 이를 수행할 수 있는 상황인가요?

**AI 응답 요약**: 과제 외부 스펙은 그대로 지키되, 내부 구조를 `SQL Processor + Page + Buffer Cache`로 설계하는 것은 충분히 가능하다고 정리했다.
**내 판단**: 채택. 차별점은 내부 구조로 가져가고, 출력/에러/CLI는 공통 스펙을 유지하는 방향이 안전하다.

### 3단계: 구현 흐름 정리
> 우선 저희의 과제입니다. 이를 파악하고 제가 제 영역에서 어떤 것을 해야하는지 흐름을 알려주세요.

**AI 응답 요약**: `입력 -> 파싱 -> 실행 -> 저장 -> 출력` 흐름과 `jihyun`이 맡아야 할 코드/문서 작업을 단계별로 정리했다.
**내 판단**: 채택. 발표와 문서 작성까지 고려하면 구현 흐름을 먼저 언어화해두는 게 좋다.

### 4단계: 문서 + 내부 구조 생성
> 문서로도 같이 남겨야 하는 부분입니다. 문서로도 남겨주시고 지금 member-jihyun 폴더에 구현하면 좋은 내부 구조를 만들어주세요.

**AI 응답 요약**: `decisions.md`, `prompts.md`, `trouble.md`, `docs/internal-architecture.md`를 채우고, `parser / executor / schema / storage / page / buffer_cache` 기준의 C 프로젝트 구조를 제안 및 생성했다.
**내 판단**: 채택. 이번 과제는 설명 가능한 구현이 중요하므로, 문서와 코드 구조를 함께 남기는 방식이 가장 효율적이다.

### 5단계: Public 테스트 실행 요청
> common/test/public의 테스트케이스를 실행해주시고, 통과 및 실패를 알려주세요.

**AI 응답 요약**: `member-jihyun/src/sql_processor`를 빌드한 뒤 `common/tests/public` 테스트를 실행했고, 총 6개 테스트가 모두 PASS(100%)임을 보고했다.
**내 판단**: 채택. 구현 상태를 빠르게 검증하고, 현재 기준에서 기능이 정상 동작함을 확인할 수 있었다.

### 6단계: 체크리스트 기반 단계별 검증 요청
> 체크리스트의 1번 ~ 7번까지 순서대로 확인하고 성공 시 체크해주세요.

**AI 응답 요약**: 항목별로 실제 실행 검증을 수행하고, 성공 시 `project-checklist.md`에 즉시 체크를 반영했다. 중간에 실패 원인이 테스트 입력(스키마 불일치)인 것을 분리해 재검증 후 정상 통과 처리했다.
**내 판단**: 채택. 구현 검증을 기능 단위로 쪼개 진행하니 현재 상태를 명확하게 파악할 수 있었고, 제출 전 불확실성을 줄일 수 있었다.
